# 前端设计技能

当交付质量取决于艺术指导、信息层级、克制表达、图像叙事与动效，而非组件数量时，使用此技能。

目标：产出有明确意图、质感高级、符合当代审美的界面。默认采用获奖级构图：一个核心概念、强视觉图像、精简文案、严格间距，以及少量但难忘的动效。

## 工作模型

开始制作前，先写清三件事：

- 视觉主张：用一句话描述氛围、材质感和能量
- 内容规划：首屏、支撑信息、细节深化、最终 CTA
- 交互主张：2-3 个能改变页面气质的动效想法

每个区块只承担一个职责、一个主导视觉概念，以及一个核心结论或动作。

## 高质量默认项

- 从构图开始，而不是从组件开始。
- 优先使用全出血首屏或全画布视觉锚点。
- 让品牌或产品名成为最醒目的文本。
- 文案保持足够精简，能在几秒内扫读完。
- 先用留白、对齐、尺度、裁切、对比，再考虑加界面装饰。
- 约束系统复杂度：最多两种字体，默认一种强调色。
- 默认不使用卡片式布局，改用分区、栏位、分隔线、列表和媒体区块。
- 把首屏当作海报，而不是文档。

## 落地页

默认结构顺序：

1. 首屏：品牌或产品、核心承诺、CTA、一个主导视觉
2. 支撑：一个具体功能、优惠或证明点
3. 细节：氛围、流程、产品深度或故事
4. 最终 CTA：转化、开始、访问或联系

首屏规则：

- 只保留一个构图方案。
- 使用全出血图像或主导视觉平面。
- 全出血规范：在品牌型落地页中，首屏本体必须真正顶边到顶边，不继承页面左右内边距、不放进框式容器、不共享 max-width；只约束内部文案/动作列宽。
- 层级顺序：品牌第一，标题第二，正文第三，CTA 第四。
- 默认不要首屏卡片、数据条、Logo 云、标签堆砌或悬浮仪表盘。
- 桌面端标题尽量控制在 2-3 行，移动端一眼可读。
- 文案列保持偏窄，并锚定在图像中相对平静的区域。
- 图像上的文字必须具备高对比度与清晰可点按目标。

如果去掉图像后首屏依然成立，说明图像太弱。若隐藏导航后品牌感消失，说明层级太弱。

首屏预算：

- 若首屏包含 sticky/fixed 顶栏，顶栏高度应计入首屏预算。常见桌面与移动尺寸下，顶栏 + 首屏内容必须在初始视口内完整呈现。
- 使用 `100vh`/`100svh` 首屏时，应扣除持久 UI 占位（`calc(100svh - header-height)`），或让顶栏覆盖其上，而不是在普通文档流中叠加。

## 应用界面

默认采用 Linear 风格的克制表达：

- 平静的界面层级
- 强调字体与间距系统
- 更少的颜色
- 信息密度高但易读
- 最小化界面装饰
- 仅当卡片本身是交互对象时才使用卡片

应用 UI 建议围绕以下结构组织：

- 主工作区
- 导航区
- 次级上下文或检查面板
- 一个清晰的强调色用于动作或状态

避免：

- 仪表盘式卡片拼贴
- 每个区域都加粗边框
- 在常规产品 UI 后方铺装饰性渐变
- 多个相互竞争的强调色
- 无助于快速扫描的装饰图标

如果某个面板改成普通布局后不损失语义，就去掉卡片样式。

## 图像

图像必须承担叙事任务。

- 在品牌、场地、编辑内容和生活方式产品中，至少使用一张有力度且真实感强的图像。
- 优先使用场景化摄影，而不是抽象渐变或伪 3D 物体。
- 选择或裁切出适合承载文字的稳定明暗区域。
- 不要使用含有路牌、Logo 或与界面冲突文字杂讯的图像。
- 不要生成自带 UI 外框、分栏、卡片或面板的图像。
- 若需要表达多个时刻，请用多张图，而不是拼贴合成一张。

首屏必须有真实的视觉锚点，仅靠装饰纹理不够。

## 文案

- 使用产品语言写作，而不是设计评论语气。
- 让标题承载核心含义。
- 支撑文案通常保持为一句短句。
- 删除区块之间的重复表达。
- 不要把提示词语气或设计点评写进 UI。
- 每个区块只承担一种责任：解释、证明、深化或转化。

如果删掉 30% 文案后页面更好，就继续删。

## 产品 UI 的功能性文案

当交付对象是仪表盘、应用工作区、后台工具或运营界面时，默认采用功能性文案，而不是营销文案。

- 优先传达定位、状态和动作，而非承诺、情绪或品牌腔调。
- 先从工作表面开始：KPI、图表、筛选器、表格、状态或任务上下文。除非用户明确要求，否则不要加入首屏 Hero 区。
- 区块标题应直接说明该区域是什么，或用户能在这里做什么。
- 正例："已选 KPI"、"计划状态"、"搜索指标"、"Top 分群"、"最近同步"。
- 若无明确需求，避免在产品界面中使用愿景式 Hero 文案、隐喻、活动传播语和高管摘要横幅。
- 支撑文本应在一句话内说明范围、行为、新鲜度或决策价值。
- 如果一句话看起来像首页 Hero 或广告语，把它改写到像产品 UI 文案为止。
- 如果某个区块不能帮助用户操作、监控或决策，就删除它。
- 快速自检：操作者只看标题、标签和数字时，能否立即理解页面？

## 动效

使用动效是为了建立存在感与层级，而不是制造噪音。

对于视觉驱动的页面，至少提供 2-3 个有目的的动效：

- 一个首屏入场序列
- 一个与滚动联动、吸附或景深相关的效果
- 一个强化可感知性的悬停、揭示或布局过渡

可用时优先使用 Framer Motion 实现：

- 区块揭示动画
- 共享布局过渡
- 滚动联动的透明度、位移或缩放变化
- 吸附式叙事滚动
- 推进叙事而非单纯占位的轮播
- 菜单、抽屉与弹窗的存在感动效

动效规则：

- 在短录屏里也能被明显感知
- 在移动端保持流畅
- 快速且克制
- 全页保持一致性
- 仅装饰性的动效应删除

## 硬性规则

- 默认不使用卡片。
- 默认不使用首屏卡片。
- 当需求要求全出血时，不要做成盒式或中栏首屏。
- 每个区块不超过一个主导概念。
- 任何区块都不应依赖大量细碎 UI 小装置来完成表达。
- 在品牌型页面里，标题不能压过品牌。
- 不写填充型废话。
- 除非文字落在统一且平静的一侧，否则不要做分屏首屏。
- 没有明确理由时，字体不超过两种。
- 除非产品已有强系统，否则强调色不超过一种。

## 这些失败方案应直接否定

- 首印象是通用 SaaS 卡片网格
- 图像很美但品牌存在感很弱
- 标题很强但没有明确动作
- 文字背后图像过于杂乱
- 多个区块重复同一种情绪表达
- 轮播没有叙事目的
- 应用 UI 靠堆叠卡片而不是布局组织

## 快速检验

- 在首屏中，品牌或产品是否一眼可辨？
- 是否存在一个强视觉锚点？
- 仅扫读标题就能理解页面吗？
- 每个区块是否只做一件事？
- 卡片真的有必要吗？
- 动效是否提升了层级或氛围？
- 如果去掉所有装饰性阴影，设计仍然高级吗？
